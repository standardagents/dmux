Dashboard Mode: Wheel Layout

Panes flow through a fixed grid geometry — no resize churn

The Problem

Current behavior: every pane add/remove triggers a full grid recalculation. Column count changes, widths shift, everything resizes. In dashboard mode with frequent expand/collapse, this would be constant visual disruption.

The Wheel Concept

Fixed grid slots. Panes needing attention queue into them. User focuses on top-left; new arrivals appear bottom-right and flow up/left as earlier panes are resolved.

Key Properties

  • Fixed geometry — Grid is always 2x2 (or configurable). No resize when panes expand/collapse.
  • Stable focus point — Slot 1 (top-left) is always the primary attention target.
  • Queue semantics — New arrivals go to the lowest empty slot. When slot 1 is resolved, everything shifts up/left.
  • Empty slots are fine — If only 1 pane needs attention, slots 2-4 stay empty (spacer panes or blank).
  • Overflow — If more than 4 panes need attention, extras stay minimized in sidebar with attention markers. They auto-fill slots as they free up.

Open Questions

A

Fixed 2x2 grid (4 slots)

Always 2 columns, 2 rows. Simple, predictable. Handles up to 4 attention panes before overflow.

B

Fixed 2 columns, rows expand

Always 2 columns but rows grow as needed (2x1, 2x2, 2x3...). More capacity, slight height redistribution when a new row is added.

C

Adaptive with max slots setting

User configures max visible slots (default 4). Grid geometry adjusts once at dashboard toggle, then stays fixed until the user changes it. Best of both worlds.
